◼︎ wp-login 페이지 접속시 자동 이동 기능이 필요한 이유

워드프레스(WordPress)에서 로그인 페이지는 기본적으로 ~/wp-login/ 또는 ~/wp-login.php 로 설정되어 있습니다.

일반 사용자에 대한 로그인 및 회원 가입 기능을 특별히 외부에 노출 및 활성화하지 않았다면, 기본 로그인 페이지는 어차피 관리자(어드민)만 사용할테니 특별한 추가 기능이 필요하지 않다고 느껴질 수 있습니다.

하지만 운영 중인 사이트에 일반 사용자를 위한 로그인/회원 가입 기능이 없더라도, 워드프레스 사이트를 전문적으로 타겟으로 하는 “무작위 대입 공격(Brute-force Attack)”을 방어하고 관리자 계정 탈취를 방지하기 위해, 항상 로그인 페이지 접속에 대한 별도의 권한 설정을 해 주는 것이 좋습니다.

또한 만일 워드프레스로 SaaS(Software as a Service) 형태의 웹어플리케이션이나 커뮤니티 사이트를 구축하고자 한다면, 일반 사용자들도 로그인 페이지를 이용해야 하기 때문에, 이렇게 기본적으로 설정된 워드프레스 로그인 페이지는 사용성과 디자인 측면에서 부적절할 수 있습니다.

이러한 상황을 고려하여, 사용자가 워드프레스 사이트의 기본 로그인 페이지(‘~/wp-login/‘)에 접속했을 때, 해당 사용자의 권한에 따라 홈(Home) 페이지 또는 별도로 만든 커스텀 로그인 페이지(Custom Login Page)로 간편하게 자동 이동(리디렉션, Redirection)시킬 수 있는 방법이 있습니다.
